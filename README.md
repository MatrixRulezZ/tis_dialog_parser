# TIS-Dialog Telegram Bot

Telegram-бот для работы с личным кабинетом провайдера **Тис-Диалог** (Калининград).

Бот позволяет нескольким пользователям удобно получать информацию из личного кабинета, не заходя на сайт.

## Возможности

- **Подробный статус** подключения (тариф, баланс, скорость, остаток турбо-трафика, IP)
- **QR-код** для быстрой оплаты через СБП
- **История платежей** (последние операции)
- **Уведомления** из личного кабинета (просмотр списка + полный текст уведомления)
- **Автоматические уведомления** при появлении новых сообщений в личном кабинете
- **Обещанный платёж** (проверка доступности и активация)
- **Фоновая проверка** баланса и IP-адреса с уведомлениями
- Поддержка **нескольких пользователей** (каждый работает под своей учётной записью)

## Технологии

- Python 3.11
- `pyTelegramBotAPI` (AsyncTeleBot)
- `aiohttp` + `BeautifulSoup4`
- SQLite
- Docker + docker-compose

## Запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/MatrixRulezZ/tis_dialog_parser.git
cd tis_dialog_parser
```

### 2. Настройка переменных окружения

Создай файл `.env` в корне проекта:

```env
BOT_TOKEN=твой_токен_бота
```

> Токен можно получить у [@BotFather](https://t.me/BotFather)

### 3. Запуск через Docker

```bash
docker-compose up -d --build
```

После запуска бот будет доступен в Telegram.

### 4. Проверка логов

```bash
docker logs -f tis_dialog_parser_tis-bot_1
```

## Использование бота

После запуска бота нажми **`/start`** и подключи свою учётную запись Тис-Диалог.

### Доступные кнопки меню:

| Кнопка                  | Описание                              |
|-------------------------|---------------------------------------|
| **Статус**              | Подробная информация о подключении    |
| **Оплатить**            | QR-код для пополнения баланса         |
| **Уведомления**         | Список уведомлений + просмотр полного текста |
| **История платежей**    | Последние операции по счёту           |
| **Обещанный платёж**    | Активация услуги при отрицательном балансе |
| **Обновить данные**     | Принудительное обновление статуса     |

## Особенности

- Бот работает в асинхронном режиме
- Использует реальный парсинг страниц личного кабинета (без публичного API)
- Сохраняет данные пользователей в SQLite
- Автоматически проверяет новые уведомления и изменения баланса/IP

## Безопасность

- Пароли пользователей хранятся в локальной базе данных
- Бот не передаёт данные третьим лицам
- Рекомендуется запускать только на своём сервере
